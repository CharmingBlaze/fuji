# Fuji compiler architecture (LLVM only)

## Pipeline

```
Fuji source (.fuji)
    |
    v
[Loader]          internal/parser     — imports, bundle
    |
    v
[Lexer]           internal/lexer      — tokenize
    |
    v
[Parser]          internal/parser     — AST
    |
    v
[Sema]            internal/sema       — analysis, native emit prep
    |
    v
[LLVM codegen]    internal/codegen    — llir/llvm → .ll
    |
    v
[Native link]     internal/nativebuild — llc / clang / libfuji_runtime.a
```

**`fuji run`** uses the same lowering as **`fuji build`**, then executes the resulting binary from a temp path.

---

## Internal layout (this repo)

| Area | Notes |
|------|--------|
| `internal/parser` | AST, recursive-descent parser, program loading |
| `internal/sema` | Semantic passes, `PrepareNativeBundle` for codegen |
| `internal/codegen` | LLVM IR construction; runtime `declare`s in `runtime.go` |
| `internal/nativebuild` | Write IR, invoke toolchain, link C runtime archive |
| `runtime/src` | NaN-boxing (`value.h`), GC, `fuji_*` runtime |

---

## Value representation

At runtime, values are **NaN-boxed `uint64_t`** (see `runtime/src/value.h`). The compiler emits **i64** for all Fuji values crossing LLVM IR boundaries.

---

## Native ABI (extern bindings)

Wrappers generated by **wrapgen** use the argv-style C ABI:

```c
Value symbol(int arg_count, Value* args);
```

Declared in LLVM as `i64 @symbol(i32, i64*)` and wired from `// fuji:extern` in `.fuji`.

---

## LLVM integration

- IR construction: `github.com/llir/llvm` (pure Go).
- Assembly / linking: **clang** (and optionally **llc**) resolved via `internal/fujihome`.
